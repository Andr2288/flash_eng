# FlashEng API Documentation

## 1. API як контракт

### Роль API у системі

API є **внутрішнім (internal)** і використовується виключно для взаємодії між frontend-застосунком (SPA на React) та зовнішніми сервісами (Supabase, OpenAI).

API орієнтований на:

- frontend (основний клієнт);
- потенційно mobile-клієнт (у майбутньому);
- не призначений для публічного (public) доступу або third-party інтеграцій.

### Призначення API

API виконує роль контракту між frontend та backend/сервісами і забезпечує:

- керування словником користувача;
- генерацію вправ за допомогою OpenAI;
- перевірку відповідей користувача;
- оновлення статусу навчальних елементів;
- отримання статистики прогресу.

API гарантує узгодженість структури запитів і відповідей, що дозволяє frontend незалежно розвиватися без зміни логіки взаємодії.

## 2. API-first дизайн

### Основні ендпоінти

#### Vocabulary API

POST /api/vocabulary  
→ додати нове слово

GET /api/vocabulary  
→ отримати список слів

GET /api/vocabulary/{id}  
→ отримати слово за ID

PATCH /api/vocabulary/{id}  
→ оновити слово або статус

DELETE /api/vocabulary/{id}  
→ видалити слово

---

#### Exercises API

POST /api/exercises/generate  
→ згенерувати вправу (через OpenAI)

POST /api/exercises/check  
→ перевірити відповідь користувача

## Swagger UI

Swagger UI використовується для інтерактивного перегляду API.

Він дозволяє:

- переглядати всі ендпоінти;
- виконувати тестові запити;
- перевіряти структуру API без написання коду.

Swagger UI був згенерований на основі OpenAPI специфікації.
