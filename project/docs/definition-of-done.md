# Definition of Done (Documentation)

## Типи змін

У проєкті FlashEng Definition of Done застосовується для таких типів змін:

- зміна API
- додавання нового ендпоінта
- зміна UI-компонента
- нефункціональні зміни

---

## Таблиця правил Definition of Done

| Тип зміни             | Обов’язкові дії                   | Перевірка          |
| --------------------- | --------------------------------- | ------------------ |
| Додано API-ендпоінт   | Оновити OpenAPI специфікацію      | OpenAPI validation |
| Змінено response      | Оновити Swagger UI                | Manual / CI        |
| Додано UI-компонент   | Додати або оновити Storybook опис | Storybook build    |
| Зміна UI-компонента   | Оновити Stories (стани)           | Storybook build    |
| Нефункціональні зміни | Оновити документацію (за потреби) | Review             |
| Breaking change       | Створити нову версію документації | Version tag        |

---

## Документаційні артефакти

Для кожного типу змін повинні бути оновлені:

- OpenAPI специфікація (docs/api/openapi.yaml)
- Swagger UI
- UI документація (Storybook)
- загальний сайт документації (MkDocs)

---

## Автоматизовані перевірки

Перед завершенням змін повинні пройти:

- валідація OpenAPI (schema validation)
- збірка Swagger UI
- збірка Storybook
- збірка документації (MkDocs)

---

## Критерії готовності

Зміна вважається завершеною, якщо:

### Що має існувати

- оновлена документація
- актуальна OpenAPI специфікація
- Swagger UI
- UI документація (Storybook)

### У якому вигляді

- у форматі Markdown / YAML
- доступна у репозиторії
- згенерована у вигляді сайту

### Де перевіряється

- у CI/CD pipeline
- у Swagger UI
- у MkDocs сайті

---

## Висновок

Документація є обов’язковою частиною Definition of Done. Жодна зміна не вважається завершеною без оновлення відповідної документації та проходження автоматичних перевірок.
